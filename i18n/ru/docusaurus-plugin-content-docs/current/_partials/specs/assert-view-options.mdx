<table>
    <thead>
        <tr>
            <td>**Опция**</td>
            <td>**Тип**</td>
            <td>**Описание**</td>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>ignoreElements</td>
            <td>`string | string[]`</td>
            <td>
                Элементы (задаются как селекторы), которые будут проигнорированы при сравнении
                скриншотов. Игнор реализуется с помощью закраски перечисленных элементов черным цветом.
                Если по любому из селекторов найдено несколько элементов, будут проигнорированы все найденные элементы.
            </td>
        </tr>
        <tr>
            <td>tolerance</td>
            <td>`number`</td>
            <td>Чувствительность к разнице в цветах, по умолчанию `3.0`. [Подробнее](https://github.com/gemini-testing/looks-same).</td>
        </tr>
        <tr>
            <td>antialiasingTolerance</td>
            <td>`number`</td>
            <td>Чувствительность к антиалиасингу, по умолчанию `4`. [Подробнее](https://github.com/gemini-testing/looks-same).</td>
        </tr>
        <tr>
            <td>allowViewportOverflow</td>
            <td>`boolean`</td>
            <td>
                По умолчанию Testplane выводит предупреждение, если элемент не удалось снять целиком.
                Эта опция отключает предупреждение.

                Используйте ее вместе с `captureElementFromTop`, чтобы полностью отключить скролл перед снятием скриншота.
            </td>
        </tr>
        <tr>
            <td>captureElementFromTop</td>
            <td>`boolean`</td>
            <td>
                По умолчанию Testplane может проскроллить страницу к элементу перед снятием скриншота,
                если элемент изначально не виден.

                Если отключить эту опцию и элемент изначально не виден, Testplane выбросит ошибку.
            </td>
        </tr>
        <tr>
            <td>compositeImage</td>
            <td>`boolean`</td>
            <td>
                По умолчанию Testplane снимает элемент целиком: делает несколько скриншотов,
                скроллит элемент и затем склеивает изображения.

                Если отключить эту опцию, будет снята только та часть элемента, которая видна на первом скриншоте.
            </td>
        </tr>
        <tr>
            <td>screenshotDelay</td>
            <td>`number`</td>
            <td>
                Задержка в миллисекундах перед снятием скриншота.

                Может пригодиться, чтобы дождаться исчезновения скроллбара перед снятием скриншота.
            </td>
        </tr>
        <tr>
            <td>disableAnimation</td>
            <td>`boolean`</td>
            <td>
                Отключение анимаций и переходов при снятии скриншота. По умолчанию `true` начиная с
                версии `8.0.0`
            </td>
        </tr>
        <tr>
            <td>ignoreDiffPixelCount</td>
            <td><code>number | \`$\{number\}%\`</code></td>
            <td>
                Процент пикселей, которые нужно игнорировать при диффе. Удобно для игнорирования
                очень маленьких диффов.

                По умолчанию `0`. Доступен начиная с версии `8.2.0`.
            </td>
        </tr>
        <tr>
            <td>waitForStaticToLoadTimeout</td>
            <td>`number`</td>
            <td>
                По умолчанию `5000`. Таймаут для команды `waitForStaticToLoad`, которая автоматически
                вызывается перед снятием скриншота.

                Если указать `0`, ожидание будет выключено.
                Доступно начиная с версии `8.32.0`.
            </td>
        </tr>
        <tr>
            <td>disableHover</td>
            <td>`"always" | "never" | "when-scrolling-needed"`</td>
            <td>
                По умолчанию Testplane отключает hover-эффекты, если для полного скриншота элемента нужен скролл.

                Доступно начиная с версии `9.0.0`.
            </td>
        </tr>
        <tr>
            <td>cropMargins</td>
            <td>`{ top?: number, right?: number, bottom?: number, left?: number }`</td>
            <td>
                Задает отступы, на которые нужно обрезать скриншот. Удобно, например, чтобы обрезать скроллбары на Android-эмуляторах.

                Доступно начиная с версии `9.0.0`.
            </td>
        </tr>
        <tr>
            <td>selectorToScroll</td>
            <td>`string`</td>
            <td>
                **Устарело**. Начиная с Testplane 9, элемент для скролла определяется автоматически.

                Ручная настройка влияет на то, какой элемент Testplane будет скроллить при снятии скриншота.
            </td>
        </tr>
    </tbody>

</table>
